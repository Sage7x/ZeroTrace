# Support

## Support Model

ZeroTrace is an open-source project with best-effort maintainer support. There is no commercial SLA implied by this repository.

## Where To Ask For Help

1. Use the issue tracker for bugs, reproducible defects, packaging problems, and documentation issues.
2. Use project discussions, if enabled, for questions, design proposals, and usage guidance.
3. Use the process in [SECURITY.md](/Users/pramodhkumars/GolandProjects/ZeroTrace/SECURITY.md) for vulnerabilities or sensitive security reports.

## What To Include In A Bug Report

1. ZeroTrace version or commit
2. OS, distro, kernel version, and architecture
3. config excerpt with secrets removed
4. expected behavior
5. observed behavior
6. relevant logs or alerts with sensitive details redacted

## Current Support Boundaries

1. Linux-first only
2. pre-`1.0` compatibility may change
3. optional control-plane features are design-stage only unless explicitly implemented
4. maintainers may ask for a minimal reproduction before investigating

## Triage Expectations

Maintainers will usually prioritize:

1. security issues
2. correctness and data-loss bugs
3. collector coverage failures on supported targets
4. high false-positive or false-negative issues
5. documentation gaps blocking contributors or operators
